## Goals

- Regenerate the live fixture ladders after the `sflo:ArtifactResolutionSpec` / `observedArtifactResolutionSpec` ontology and runtime change.
- Remove or sharply shrink Weave's temporary fixture RDF normalization bridge once live fixture branches carry the new vocabulary directly.
- Keep the fixture repositories, Semantic Flow Framework conformance examples, and Weave fixture-backed tests aligned so local and CI runs read the same generated bytes.
- Preserve the fixture-ladder model: generated fixture branch refs are disposable golden outputs, not hand-edited source material.

## Summary

The Artifact Resolution Spec cleanup changed source-binding RDF shape across Weave output:

- `sflo:ArtifactResolutionTarget` became `sflo:ArtifactResolutionSpec`
- old coordinate predicates such as `hasTargetArtifact`, `hasRequestedTargetState`, and `hasTargetRepositorySource` became direct `target*` predicates
- observed resolution coordinates moved from mirrored observation predicates into nested `sflo:observedArtifactResolutionSpec`
- enum-only fallback policy was replaced by recursive `sflo:hasFallbackArtifactResolutionSpec`

The Weave runtime currently has a transitional `tests/support/fixture_normalization.ts` bridge so tests can still consume stale fixture branches while the ontology/runtime commits are being reviewed. That bridge is useful for sequencing, but it should not become a compatibility layer. This task realigns the generated fixture branches with the new live vocabulary, then removes the bridge paths that only exist for stale artifact-resolution fixture RDF.

## Discussion

Fixture ladders are a cross-repo generated-output surface. The live fixture repositories are:

- `dependencies/github.com/semantic-flow/mesh-alice-bio`
- `dependencies/github.com/semantic-flow/mesh-sidecar-fantasy-rules`
- `dependencies/github.com/semantic-flow/mesh-branch-fantasy-rules`

The operational playbook is [[wd.testing.fixture-ladder-regeneration]]. This task should follow that playbook rather than inventing a bespoke rerun process.

The change is vocabulary-wide and appears early enough in source registries, reference catalogs, ResourcePage source summaries, and woven pages that targeted late-rung regeneration is likely to leave mixed old/new fixture state. Whole live-ladder reruns are the safer default.

The live ladder branch series is the `a.*` series. Older unprefixed branches are historical and should not be regenerated unless a separate task explicitly asks for archival branch repair.

Semantic Flow Framework conformance JSON has already been mechanically updated once during the ontology/runtime pass. After the live fixture rerun, verify whether those conformance examples still match the regenerated fixture outputs. If they drift, refresh them from the regenerated outputs rather than hand-editing individual old predicates.

This task is also a good time to verify that old artifact-resolution vocabulary no longer appears in fixture repos except where intentionally documented as retired vocabulary in notes, release history, or guardrail tests.

## Decisions

- Rerun all three live `a.*` fixture ladders in fixture-plan order.
- Do not hand-edit generated fixture branch contents.
- Move `main` in `mesh-alice-bio` and `mesh-sidecar-fantasy-rules` to their final live rung after review, following [[wd.testing.fixture-ladder-regeneration]].
- Treat `mesh-branch-fantasy-rules` `gh-pages` as generated publication output. After the branch-published rerun, review the generated publication result and push/update `gh-pages` deliberately; do not frame this as needing a fast-forward from a hand-maintained publication branch.
- Remove transitional artifact-resolution vocabulary normalization from Weave tests after regenerated fixture branches are available locally.
- Remove legacy compatibility cruft wherever it is encountered in Weave/SFF fixture surfaces, unless a specific historical note or retired-vocabulary guardrail intentionally needs to mention it.
- Keep old vocabulary guardrails in SFLO tests; those tests intentionally mention retired names and are not fixture drift.
- Prefer the cleaner SFF conformance path: refresh conformance examples from regenerated fixture output when possible, and only hand-adjust examples when there is no generated source of truth for that file.
- Keep this task separate from config policy implementation. Config policy should build on `sfcfg:ConfigSource` after fixture baselines are stable.

## Contract Changes

- Live fixture RDF should use `sflo:ArtifactResolutionSpec` and the new direct target predicates.
- Source registry observations should serialize concrete observed coordinates through `sflo:observedArtifactResolutionSpec`.
- Reference catalog fixture RDF should use `ReferenceSource` with `targetArtifact` and optional `targetHistoricalState`.
- Page-definition/page-source fixture RDF should use the new target predicate names and `hasFallbackArtifactResolutionSpec` when fallback is modeled.
- Fixture-backed Weave tests should no longer depend on runtime normalization of old artifact-resolution vocabulary.
- Semantic Flow Framework conformance examples should not assert the retired artifact-resolution target or mirrored observation shapes.

- Inspect generated fixture diffs for unrelated files, runtime logs, `.weave/`, editor files, or host-local paths.

## Non-Goals

- Do not implement config policy binding/runtime behavior in this task.
- Do not redesign fixture-ladder tooling unless regeneration reveals a blocking bug.
- Do not regenerate historical unprefixed fixture branch series.
- Do not preserve long-lived compatibility aliases for pre-v1 artifact-resolution vocabulary.
- Do not hand-edit generated fixture branch contents to make tests pass.
- Do not change URPX dogfood examples unless a fixture-ladder issue directly reveals a reusable CLI documentation problem.
- Do not preserve stale compatibility helpers just because they are convenient during the transition; call out any questionable remnants and remove them when practical.
