## Summary

`src/core/weave/weave.ts` still contains a bounded cluster of helpers that build `ResourcePageModel[]` values for planned weave slices. These helpers do not render HTML; they only describe which resource pages should be generated by downstream page rendering.

This slice should move those model builders into a focused module while keeping the existing planner calls and generated outputs stable. It is a good follow-up to the ReferenceCatalog current-link extraction because the builders now consume a parser result from `reference_catalog_links.ts` but do not need to own that parser logic.
